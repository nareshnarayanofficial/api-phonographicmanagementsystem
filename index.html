Getting Started

Welcome to the PhonoGraph API. Our platform enables labels and distributors to programmatically manage catalogs, issue takedowns, and calculate complex royalty splits in real-time.

Introduction

The API is organized around REST. Our API has predictable resource-oriented URLs, accepts form-encoded request bodies, returns JSON-encoded responses, and uses standard HTTP response codes, authentication, and verbs.

You can use the API in test mode, which does not affect your live data or interact with DSPs (Spotify, Apple Music). The API key you use to authenticate the request determines whether the request is live mode or test mode.

Security Warning

Your API key carries many privileges, so be sure to keep it secure! Do not share your secret API keys in publicly accessible areas such as GitHub, client-side code, and so forth.

Interactive Console

The PhonoGraph API powers our web interface. You can view API requests initiated by the web interface and use these requests as working samples to speed up your development.

Base URLs

All URLs referenced in the documentation have the following base:

https://api.phonograph-system.com/v1

Authentication

We use Bearer tokens for authentication. Pass your token in the standard Authorization header. 

curl -X GET https://api.phonograph-system.com/v1/artists \
  -H "Authorization: Bearer YOUR_API_KEY"
